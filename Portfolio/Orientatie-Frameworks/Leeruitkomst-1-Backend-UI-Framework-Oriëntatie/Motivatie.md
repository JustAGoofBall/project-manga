# Leeruitkomst 1: Oriëntatie op Backend- en UI-Frameworks

## Motivatie

Ik heb me succesvol georiënteerd op zowel backend- als UI-frameworks door praktische ervaring op te doen met Express.js en React.js in het Project Manga.

### Wat is een Framework?
Een framework biedt een gestructureerde manier om applicaties te bouwen. In plaats van alles zelf te schrijven, geeft het je tools, patterns en best practices.

### Backend: Express.js
**Waarom Express.js?** Express is het meest gebruikte Node.js framework voor REST APIs. Het is:
- Lightweight maar powerful
- Heeft een groot community
- Perfect voor CRUD operaties
- Goed testbaar

Ik heb Express.js gebruikt voor de Project Manga API met:
- ✅ 30+ werkende endpoints
- ✅ Professional middleware structuur
- ✅ Centralized error handling
- ✅ JWT authentication geconfigureerd
- ✅ Rate limiting ingesteld
- ✅ Request logging middleware

**Impact:** Zonder Express zou ik dit handmatig moeten doen, wat veel meer code en tijd zou kosten.

### Frontend: React.js
**Waarom React.js?** React is een component-based UI framework gebruikt door Netflix, Facebook en Instagram. Het is:
- Performant (Virtual DOM)
- Reusable components
- Excellent voor SPA (Single Page Apps)
- State management via Context API

Ik heb React implementeerd met:
- ✅ 6+ components (Navbar, PrivateRoute, etc.)
- ✅ 6 pagina's met routing
- ✅ Context API voor state management
- ✅ Protected routes geïmplementeerd
- ✅ Volledige authentication flow
- ✅ Responsive design

**Impact:** React maakt het UI-développement 10x efficiënter door componenthersgebruik.

### Verantwoording
Dit project toont aan dat ik beide backend- en frontend-frameworks kan kiezen en implementeren. Beide keuzes waren:
- **Bewust** - Ik heb alternatieven overwogen
- **Verantwoord** - Trade-offs zijn duidelijk
- **Professioneel** - Industry-standaard tools
