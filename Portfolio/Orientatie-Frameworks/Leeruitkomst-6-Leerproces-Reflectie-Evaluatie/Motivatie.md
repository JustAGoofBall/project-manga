### Wat Was Lastig / Moeilijk

**Uitdaging 1: Initial Architecture**
- First version was overengineered
- Refactored 2-3 times
- Lesson: Plan architecture before coding

**Uitdaging 2: React Hooks**
- useEffect dependencies confusing initially
- Re-renders performance issues
- Solution: Read React docs deeply, used DevTools

**Uitdaging 3: Testing**
- Jest/Supertest syntax unfamiliar
- Async testing tricky
- Solution: Built tests incrementally with documentation

**Uitdaging 4: Database Relations**
- Foreign keys and cascading deletes
- Relationship queries complex
- Solution: Studied MongoDB & SQL patterns

### Hoe Ik Dit Opging / Verbeterd Heb

| Problem | Solution | Result |
|---------|----------|--------|
| Complex architecture | Refactored after research | Clean MVC |
| React confusion | Studied docs + small projects | Confident with hooks |
| Testing difficulty | Built tests incrementally | 60+ passing tests |
| Database design | Studied patterns | Clean schema with relations |

### Wat Ik Anders Zou Doen

**Verbetering 1: Better Planning**
- ❌ Started coding without full design
- ✅ Should: Spend 2-3 hours on architecture first
- ✅ Future: Make technical design doc before coding

**Verbetering 2: Test-Driven Development (TDD)**
- ❌ Wrote tests after implementation
- ✅ Should: Write test FIRST, then code
- ✅ Future: All new features follow TDD

**Verbetering 3: Documentation During Development**
- ❌ Documented at the end
- ✅ Should: Document as you build
- ✅ Future: Fresh memory = better docs

**Verbetering 4: Git Practices**
- ❌ Commits sometimes too large
- ✅ Should: Atomic commits (one feature per commit)
- ✅ Future: Use proper commit messages (feat:, fix:, docs:)

**Verbetering 5: Code Review**
- ❌ Didn't get peer review
- ✅ Should: Ask for code review earlier
- ✅ Future: Regular code review process
