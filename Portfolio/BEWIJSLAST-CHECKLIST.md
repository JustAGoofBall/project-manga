# ✅ Bewijslast Checklist

Per leeruitkomst: controleer of je dit in de `bewijslast/` folder hebt.

## 🎯 Minimaal Bewijslast (per leeruitkomst)

- [ ] **Feedbackvragen.md** - Vragen met code links (in parent folder)
- [ ] **bewijslast/screenshots/** - Minstens 1-2 screenshots
  - [ ] Relevant test output of coverage
  - [ ] Code example screenshot
- [ ] **bewijslast/diagrams/** - Minstens 1 diagram
  - [ ] Architecture / flow diagram
  - [ ] Data model / schema
- [ ] **bewijslast/logs/** - Output/metrics
  - [ ] Relevante logs of test results

## 📋 Per Vak Specifiek

### API-Basis
- [ ] Test coverage screenshot (85%+)
- [ ] Endpoints list diagram
- [ ] Database schema diagram
- [ ] API request examples

### Orientatie-Frameworks  
- [ ] MVC architecture diagram
- [ ] Framework comparison screenshot
- [ ] Database schema
- [ ] Code organization examples

### Frameworks-Gevorderd
- [ ] Coverage report (80%+)
- [ ] Test results screenshot
- [ ] Security flow diagram
- [ ] Code quality metrics

---

**Werkwijze**: 
1. Maak screenshot van coverage/test
2. Save als `bewijslast/screenshots/coverage.png`
3. Maak diagram in draw.io/lucidchart
4. Save als `bewijslast/diagrams/diagram.png`
5. Copy relevante logs naar `bewijslast/logs/`
