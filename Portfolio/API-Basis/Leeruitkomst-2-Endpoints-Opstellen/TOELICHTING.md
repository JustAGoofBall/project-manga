# 📐 Leeruitkomst 2: Endpoints voor API Opstellen

## Wat Ik Toon
Ik demonstreer dat ik **goed ontworpen endpoints** kan opstellen met:
- Duidelijke request/response patterns
- Juiste HTTP status codes
- Consistent naming convention
- Proper error handling
- Request validation

**Bewijs:**
- Request/response diagrams per endpoint-type
- Code voorbeelden van endpoint implementation
- Error response voorbeelden
- Validation logica getoond

## Hoe Dit In Mijn Project Zichtbaar Is
- **Controllers:** `/controllers/` folder bevat alle business logic
- **Validatie:** `/validators/` folder bevat input validation per feature
- **Error Handling:** `/middleware/errorHandler.js` centralized error responses
- **Demonstratie:** Zie [ENDPOINT-DESIGN.md](ENDPOINT-DESIGN.md) voor design breakdowns
