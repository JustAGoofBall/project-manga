# 🏗️ Leeruitkomst 1: API's Bouwen op Basis van Endpoints

## Wat Ik Toon
Ik demonstreer dat ik een **volledige RESTful API** kan bouwen met duidelijke endpoints. Elk endpoint volgt REST-standaarden en is goed georganiseerd in routes.

**Bewijs:**
- 8+ API routes geïmplementeerd met Express.js
- Alle CRUD operaties: Create, Read, Update, Delete
- Consistent endpoint naming en structuur
- Status codes correct toegepast (200, 201, 400, 404, 500)

## Hoe Dit In Mijn Project Zichtbaar Is
- **Routes:** `/routes/` folder met `anime.js`, `characters.js`, `auth.js`, `ratings.js`, `favorites.js`, etc.
- **Endpoints:** Elk bestand defineert 3+ endpoints voor die resource
- **Demonstratie:** Zie [ENDPOINTS-SHOWCASE.md](ENDPOINTS-SHOWCASE.md) voor de volledige overzicht
